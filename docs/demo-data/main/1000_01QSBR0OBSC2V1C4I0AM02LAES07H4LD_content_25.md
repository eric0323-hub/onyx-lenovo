### Symptom

In the Hub 500 Zoom Rooms System, when an Administrator logs on, and launches the search box by touching the magnifying glass on the task bar, touching in that search box sometimes does not prompt the On Screen Keyboard (OSK) to appear.

This can happen in the following situation:

* The Administrative user is logged in: "ZoomRoomsAdmin"
* There is no other application that accepts text input currently open to have triggered the OSK
* The user clicks on the magnifying glass on the task bar in order to search

### Applicable Brands

Smart Collaboration

### Applicable Systems

Hub 500

### System Is Configured With

Zoom Rooms System

### Operating Systems

Preinstalled Windows 10 IoT Semi-Annual Channel

### Limitations

The user account does not encounter this issue since the interface for the user is the Zoom Rooms environment, not Explorer.

### Additional Information

Lenovo is working with Microsoft to develop a solution for this issue.

### Workaround

In taskbar properties (press and hold the carat symbol in the system tray), tap **Show Touch Keyboard Button** and a keyboard icon will appear in the system tray. Then open search, touch in the search box, and if the OSK does not appear, touch the keyboard icon in the tray and the system OSK will appear. Tap in your search term and it will be entered on the search bar.

Or

Open another application which accepts touch input (such as notepad, file explorer, and so on) and touch the input area for that app or utility (for example, the document for notepad, the file navigation box for file explorer) and the OSK will appear. Once this happens, the OSK will also work in the search box launched from the task bar.

Or

Attach a physical keyboard to the device.
