# 局域网通信延迟导致打印任务堆积

## 背景

客户报告多个前台终端可以正常登录系统，但提交的打印任务经常延迟 20 秒以上才到达后厨打印机。打印机本身没有报错，重启应用后问题短暂缓解，午高峰又反复出现。

## 判断方法

此类问题重点关注网络链路，而不是应用账号。现场可在同一网段电脑上连续 ping 打印机地址，并观察延迟和丢包率。如果平均延迟超过 100ms 或出现间歇性丢包，应检查交换机端口、网线和无线桥接设备。

## 处理建议

将打印机从无线连接改为有线连接，优先接入主交换机。若使用多个路由器扩展 WiFi，应关闭二级路由的 DHCP，避免终端和打印机落在不同网段。对于客流高峰门店，建议为业务终端设置固定 IP，并在路由器中保留地址，减少地址冲突。

## 验收

连续发送 20 个测试打印任务，单个任务从提交到打印机响应的时间应低于 3 秒。若仍有延迟，需要导出路由器在线设备列表和交换机端口日志，升级给网络工程师处理。
