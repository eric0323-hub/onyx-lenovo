# 服务端接口超时与 500 错误排查记录

## 背景

上午 10 点开始，多个企业客户反馈后台首页加载缓慢，偶尔出现“系统繁忙，请稍后重试”。监控显示订单统计接口 95 分位响应时间从 300ms 上升到 8 秒，少量请求返回 500。

## 排查路径

值班工程师应先确认是否为单客户数据异常。若多个租户同时受影响，需要查看接口错误率、数据库连接池占用、缓存命中率和下游服务响应时间。若 500 集中在同一个接口，应抓取 trace id 并查找对应服务日志。

## 临时处理

当接口超时来自慢查询时，可先降低首页统计范围，暂停非关键聚合任务，避免请求继续堆积。若错误来自下游报表服务，应启用降级逻辑，首页展示最近一次成功缓存的数据，并提示数据稍后刷新。

## 恢复验证

修复后观察 30 分钟，确认接口 95 分位响应时间恢复到 1 秒以内，500 错误不再增长。事故记录中需要包含影响时间、错误代码、根因、临时降级动作和后续优化项。
