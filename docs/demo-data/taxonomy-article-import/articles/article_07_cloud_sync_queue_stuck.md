# 云端同步队列卡住导致门店库存不更新

## 背景

客户发现门店端已经完成商品盘点，但总部后台仍显示前一天库存。门店网络可以访问外网，终端也在线，但同步状态一直停留在“等待上传”。

## 问题现象

本地数据库中可以看到新增盘点记录，说明业务操作已经写入设备。同步任务队列中存在大量重试记录，最新错误提示为远端版本冲突。该问题通常与多台设备同时修改同一批商品有关。

## 处理步骤

先暂停门店继续盘点，避免产生更多冲突记录。导出本地待同步队列，确认是否存在重复商品编码和相同更新时间。若只是版本冲突，可以执行一次强制拉取总部最新版本，再重新提交本地增量。

如果队列中存在损坏记录，不要直接删除本地数据。应先备份同步数据库，再由二线支持使用修复脚本重建队列索引。修复完成后，观察同步进度是否从等待上传变为上传中，并在总部后台检查库存数量。

## 复盘建议

对于高频盘点门店，建议开启冲突提示，并限制同一商品在多台设备上同时编辑。培训门店员工在断网期间不要反复提交同一批盘点结果。
