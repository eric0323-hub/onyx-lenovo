# 本地存储空间满导致历史记录无法保存

## 背景

一台售后工作站提示“保存失败，磁盘空间不足”。用户已经完成多笔维修记录录入，但点击提交后页面无响应。后台没有收到这些记录，现场担心数据丢失。

## 风险判断

存储空间不足时，应用可能无法写入日志、缓存和本地队列。若用户在失败后关闭浏览器，未提交的数据可能只保留在临时缓存中。应立即停止清理浏览器数据，避免误删待恢复内容。

## 恢复流程

先检查工作站系统盘剩余空间，删除无关安装包和旧日志，确保至少释放 2GB。然后重新打开应用，查看是否出现未提交记录恢复提示。若没有提示，导出本地缓存目录并提交给二线支持分析。

如果记录已经进入本地队列但未上传，可使用恢复工具读取队列文件，将维修单号、客户信息和处理内容导出为表格，再批量补录到后台。补录完成后，应核对每条记录的创建时间，避免重复生成工单。

## 预防

建议在工作站上开启磁盘空间告警，并定期清理临时文件。对需要长期离线操作的站点，应把本地缓存目录迁移到容量更大的分区。
