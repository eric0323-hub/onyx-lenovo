# 企业子账号只能查看所属区域数据的配置方法

## 背景

客户总部希望给华东、华南两个区域团队分别创建子账号。区域经理只能查看自己负责门店的数据，不能查看其他区域的订单、客户和库存。

## 配置原则

权限应按组织结构和数据范围同时配置。只给用户分配角色还不够，还需要把角色绑定到对应区域或门店集合。总部管理员保留全局查看权限，区域经理使用受限角色。

## 操作步骤

先在组织架构中创建华东和华南两个部门，并把对应门店加入部门。然后创建“区域经理”角色，勾选订单查看、库存查看和客户查询权限，但不授予财务结算和系统设置权限。

给子账号分配角色时，数据范围选择用户所属部门。配置完成后，用测试账号分别登录，确认华东账号无法搜索华南门店订单，华南账号也无法导出华东客户清单。

## 审计要求

每次权限变更都需要保留审批记录，包括申请人、审批人、变更原因和生效时间。离职或调岗用户应在当天禁用账号或调整部门，防止继续访问原区域数据。
