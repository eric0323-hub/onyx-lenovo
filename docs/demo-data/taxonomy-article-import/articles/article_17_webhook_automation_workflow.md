# 使用 Webhook 构建售后自动化工作流

## 背景

客户希望当高优先级售后工单创建时，自动通知企业内部 IM 群，并在 CRM 中创建跟进任务。手动复制信息效率低，且容易遗漏紧急客户。

## 配置思路

可以使用平台的 Webhook 功能监听工单创建事件。触发条件设置为优先级等于高，且工单来源为客户提交。系统会把工单编号、客户名称、问题摘要和负责人发送到客户配置的接收地址。

## 操作步骤

先在开发者设置中创建 Webhook，填写接收端 URL 和签名密钥。然后选择事件类型为工单创建，并添加过滤条件。接收端服务验证签名后，可以调用企业 IM 和 CRM API，分别发送群消息和创建任务。

## 最佳实践

建议为自动化流程设置重试和幂等逻辑。若接收端返回超时，平台会重新推送同一事件，接收端应根据事件 ID 避免重复创建 CRM 任务。上线前使用测试企业完成全链路验证。

## 安全要求

签名密钥不要写在前端代码或共享文档中。若怀疑密钥泄露，应立即轮换，并检查最近的 Webhook 调用日志。
