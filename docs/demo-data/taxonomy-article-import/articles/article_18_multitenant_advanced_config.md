# 多租户数据隔离的高级配置建议

## 背景

集团客户下属多个事业部，希望在同一平台中统一管理账号和套餐，同时保证事业部之间的数据互不可见。集团总部仍需要跨事业部查看经营报表。

## 配置模型

建议使用多租户空间承载事业部数据，每个空间绑定独立的数据范围、角色模板和默认门店。总部账号授予跨空间只读权限，事业部管理员只能管理本空间成员和业务配置。

## 实施步骤

先梳理集团、事业部和门店的层级关系，再建立租户空间。每个空间配置独立的角色模板，例如事业部运营、门店主管和客服专员。然后为总部账号配置跨空间报表权限，但不授予批量修改业务数据的能力。

## 验证方法

实施完成后，用不同角色账号分别登录，验证搜索、导出、工单查看和客户列表是否符合隔离要求。特别要测试共享客户和跨区域订单，避免因为公共字段导致数据泄漏。

## 维护建议

组织架构调整时，应先在测试空间演练权限变化，再同步到生产空间。对于临时项目组，建议设置权限到期时间，项目结束后自动回收访问范围。
