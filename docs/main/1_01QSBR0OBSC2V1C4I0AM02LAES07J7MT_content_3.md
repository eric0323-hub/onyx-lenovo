## 🧰 Troubleshooting Ethernet Port Issues on Laptops and Desktops

## 🔄 Auxiliary Hardware Check

1. **Check for Dock Usage**
   
   * **Ask the Customer:** Are they using a dock?
     
     + **Yes:** Refer to the Dock Troubleshooting Guide.[https://ukm.lenovo.com/#/homePage/homeDetail?uid=0x44f464ac&title=Dock+TS+Guide](https://ukm.lenovo.com/#/homePage/homeDetail?uid=0x44f464ac&title=Dock+TS+Guide&back=/homePage/searchDetail)
     + **No:** Continue to the next step.
2. **Modem/Router Access**
   
   * **Ask the Customer:** Do they have access to the modem or router?
     
     + **Yes:** Instruct them to reset it (if possible).
     + **Test:** Check if the issue is resolved.
       
       - **Yes:** Solution provided.
       - **No:** Continue to the next step.
3. **Ethernet Adapter Check**
   
   * **Ask the Customer:** Are they using an Ethernet adapter?
     
     + **Yes:** Confirm with the OCM if it is a supported adapter.
       
       - If not supported, recommend the correct adapter and advise them to call back if they have further issues.
     + **No:** Proceed to Step 2.

## 🧰 Cable and Connection Checks

4. **Reseat the Cable**
   
   * Instruct the customer to remove and re-insert the Ethernet cable.
   * **Issue Resolved?**
     
     + **Yes:** Solution provided.
     + **No:** Proceed to Step 3.
5. **Check for Link Lights**
   
   * **Ask the Customer:** Do they see "link lights"?
     
     + **Note:** Link lights are the small LED indicators that blink when an Ethernet connection is active.
     + **Yes:** Proceed to Step 6.
     + **No:** Proceed to Step 4.
6. **Cable and Wall Port Inspection**
   
   * Have the customer inspect the Ethernet cable for any visible damage.
   * If possible, ask them to:
     
     + Use a different Ethernet cable.
     + Use a different Ethernet jack in the wall.
7. **Reset the Laptop**
   
   * **Note:** This step does not apply to desktops. If the device is a desktop, skip to Step 5.
   * Conduct a pinhole reset. [https://ukm.lenovo.com/#/homePage/homeDetail?uid=0x44f44f6a&title=Conduct+a+Pin+Hole+Reset](https://ukm.lenovo.com/#/homePage/homeDetail?uid=0x44f44f6a&title=Conduct+a+Pin+Hole+Reset&back=/homePage/searchDetail)
   * **Issue Resolved?**
     
     + **Yes:** Solution provided.
     + **No:** Proceed to Step 6.

## 💻 Software Check

8. **Driver Version Check**
   
   * Have the customer provide the driver version of the Ethernet adapter.
   * **Where to Find:**
     
     + Device Manager → Network Adapters → Right-click on Ethernet → Properties → Driver tab → Version.

### Scenario 1: Driver Present but Out of Date

* **Solution:** Instruct the customer to connect to Wi-Fi and download the latest driver from the Lenovo support site or download it to a USB-A stick and install it on the laptop.
* **Issue Resolved?**
  
  + **Yes:** Solution provided.
  + **No:** Proceed to Step 7.

### Scenario 2: Driver Missing

* **Solution:** Same as Scenario 1.
* **Issue Resolved?**
  
  + **Yes:** Solution provided.
  + **No:** Proceed to Step 7.

### Scenario 3: Driver Present with a "Yellow" Bang

* **Solution:** Uninstall the driver, then have the customer connect to Wi-Fi and download the latest driver from the Lenovo support site or use a USB-A stick.
* **Issue Resolved?**
  
  + **Yes:** Solution provided.
  + **No:** Proceed to Step 7.

## ⚙️ Hardware Deployment

**Important Note:** Ensure to check the location of the Ethernet port (RJ45 port). Some laptop models may have the RJ-45 port on a separate sub-card.

* **Reference:** How to Determine if PARTNAME is on the System Board. [https://ukm.lenovo.com/#/homePage/homeDetail?uid=0x44f0c88a&title=HOW+TO+-+Figure+out+if+PARTNAME+is+on+the+system+board](https://ukm.lenovo.com/#/homePage/homeDetail?uid=0x44f0c88a&title=HOW+TO+-+Figure+out+if+PARTNAME+is+on+the+system+board&back=/homePage/searchDetail)
* Send either the system board or RJ45 subcard for further assistance.

By following these steps, you should be able to effectively troubleshoot and resolve Ethernet port issues on laptops and desktops. If the problem persists, consider escalating the issue to a higher level of support.
